# AGENT_PROMPTS.md
# 漫享家 / MangaHaven 给 Claude Code / 其他 coding agent 的执行提示词

## 通用执行规则

你正在开发一个 Android 原生离线漫画阅读器。  
请始终遵守以下规则：

1. 使用 Kotlin + Jetpack Compose
2. 使用分层架构，保持 UI / Domain / Data 职责清晰
3. 每次只完成一个阶段，不要擅自扩展到下个阶段
4. 优先保证代码可运行、可验证、可维护
5. 不要为了“功能看起来多”牺牲架构质量
6. 每次输出必须包括：
   - 修改了哪些文件
   - 为什么这样设计
   - 如何运行
   - 如何验收
   - 已知限制
7. 如果某功能会明显拉高复杂度，请先做可扩展抽象，再做最小实现
8. 所有 IO 和解码都不能阻塞主线程
9. 所有新增代码尽量附带基础测试
10. 不要生成在线漫画源爬虫功能

---

## Prompt 1：创建工程骨架

请为一个 Android 离线漫画阅读器创建工程骨架。

要求：
1. 使用 Kotlin + Jetpack Compose + Material 3；
2. 使用多模块结构，至少包含：
   - app
   - core-model
   - core-reader
   - data-local
   - data-files
   - feature-library
   - feature-reader
   - feature-settings
3. 接入 Room、DataStore、WorkManager；
4. 采用 ViewModel + StateFlow；
5. 提供统一 Result/Error 模型；
6. 提供书架、阅读器、设置三个页面的空壳；
7. 提供导航；
8. 代码必须可编译运行；
9. 不实现业务功能，只搭基础框架。

输出格式要求：
- 先给目录结构
- 再给关键类与职责说明
- 再给关键代码
- 最后给运行步骤与验收方式

## Prompt 2：实现 MVP 本地离线阅读闭环

请在现有工程上实现第一版可用功能，让用户能导入本地漫画并阅读。

要求：
1. 支持 Android SAF 选择单文件和目录；
2. 支持图片文件夹；
3. 支持 ZIP / CBZ；
4. 建立书架页，显示封面、标题、最近阅读；
5. 阅读器支持：
   - 左右翻页
   - 纵向滚动
   - 缩放
   - 拖拽
   - 记忆阅读进度
6. 应用重启后可恢复书架和进度；
7. 加入基础预加载和缓存；
8. 不要实现 RAR、PDF、WebDAV、SMB、OPDS；
9. 所有 IO 必须放后台线程；
10. 输出可运行代码和验收步骤。

额外要求：
- 优先抽象出 SourceClient、ContainerReader、PageProvider、ProgressRepository
- 不要把本地逻辑写死在 UI 层

## Prompt 3：增强阅读体验

请在现有阅读器基础上实现增强版阅读体验。

要求：
1. 支持三种阅读方向：
   - LTR
   - RTL
   - Vertical
2. 支持双页模式；
3. 支持白边裁切；
4. 支持页码偏移修正；
5. 支持缩略图跳页；
6. 支持每本书单独覆盖全局阅读设置；
7. 保持现有架构不被破坏；
8. 所有新增配置需可持久化；
9. 给出必要测试点。

实现时注意：
- 双页组合逻辑要可维护
- 白边裁切失败时要自动回退原图
- 不能明显影响已有性能

输出要求：
- 先说明设计方案
- 再输出关键修改文件
- 再输出测试建议
- 最后说明已知限制

## Prompt 4：接入 WebDAV

请基于现有 SourceClient 抽象，为漫画阅读器接入 WebDAV 远程源。

要求：
1. 增加“添加 WebDAV 源”的配置页面；
2. 支持保存地址、用户名、密码；
3. 支持测试连接；
4. 支持浏览远程目录；
5. 支持读取远程图片或 ZIP / CBZ；
6. 阅读时采用按需读取，不整本下载；
7. 提供基础缓存；
8. 网络失败时可重试；
9. 不要重写书架和阅读器核心；
10. 所有逻辑尽量复用现有接口。

输出要求：
- 新增了哪些接口与实现
- 如何存储连接配置
- 如何处理鉴权和错误
- 如何本地测试

## Prompt 5：接入 SMB

请基于现有 SourceClient 抽象，为漫画阅读器接入 SMB。

要求：
1. 支持配置 SMB 地址和凭证；
2. 支持浏览共享目录；
3. 支持按需读取远程文件；
4. 支持远程封面加载；
5. 支持失败重试；
6. 不重写阅读器核心；
7. 保持与 WebDAV 实现风格一致；
8. 给出测试样例和已知边界。

特别注意：
- SMB 容易在错误处理和连接管理上失控
- 请把连接生命周期管理清晰
- 不要把协议细节泄漏到 UI 层

## Prompt 8：代码审查模式

请对当前项目进行一次架构与实现审查，不新增大功能。

要求：
1. 重点检查模块边界是否清晰；
2. 检查 UI 层是否侵入数据层逻辑；
3. 检查 IO 是否错误地运行在主线程；
4. 检查阅读器是否存在 OOM 风险；
5. 检查数据库实体和 Repository 是否过度耦合；
6. 给出按优先级排序的问题列表；
7. 对每个问题给出建议修复方案；
8. 不要泛泛而谈，要结合具体文件和代码结构。

---
